# Capitalize-Expansion
This is a [PlaceholderAPI](https://links.alonsoaliaga.com/PlaceholderAPI) expansion to allow owners/configurator customize texts a bit more.

# Installation
You can install this expansion in 2 ways:
### 1) PlaceholderAPI eCloud (Available ✔️)
While being in console or having OP run the following commands:
> /papi ecloud download capitalize\
> /papi reload

✅ Expansion is ready to be used!
### 2) Manual download
Go to [eCloud](https://api.extendedclip.com/expansions/capitalize/) and click `Download Latest` button to get the .jar file.\
Copy and paste the file in `/plugins/PlaceholderAPI/expansions/` and run:
> /papi reload

✅ Expansion is ready to be used!
# Placeholders
The following placeholders are available:
> ### %capitalize_pascal-case_{THE_PLACEHOLDER}%
> Allows you to use convert the output of the placeholder using pascal case. <br>
> This means that all first letters of the parsed placeholder will be uppercase. <br>
> Supports PlaceholderAPI but requires `{ }` instead of `% %`.<br>
>
> **Example:** %capitalize_pascal-case_{worldguard_region_name}%<br>
> **Output:** As the placeholder returns the name of the WorldGuard region "the_market_WORLD" it will return: `The Market WORLD`

> ### %capitalize_pascal-case-forced_{THE_PLACEHOLDER}%
> Allows you to use convert the output of the placeholder using pascal case. <br>
> This means that all first letters of the parsed placeholder will be uppercase. <br>
> This is different from the previous one as it will FORCE lowercase in the rest of the word. <br>
> Supports PlaceholderAPI but requires `{ }` instead of `% %`.<br>
>
> **Example:** %capitalize_pascal-case-forced_{worldguard_region_name}%<br>
> **Output:** As the placeholder returns the name of the WorldGuard region "the_market_WORLD" it will return: `The Market World`

> ### %capitalize_sentence-case_{THE_PLACEHOLDER}%
> Allows you to use convert the output of the placeholder using sentence case. <br>
> This means that only the first letters of the parsed placeholder will be uppercase. <br>
> Supports PlaceholderAPI but requires `{ }` instead of `% %`.<br>
>
> **Example:** %capitalize_sentence-case_{worldguard_region_name}%<br>
> **Output:** As the placeholder returns the name of the WorldGuard region "the_market_WORLD" it will return: `The market WORLD`

> ### %capitalize_sentence-case-forced_{THE_PLACEHOLDER}%
> Allows you to use convert the output of the placeholder using sentence case. <br>
> This means that only the first letters of the parsed placeholder will be uppercase. <br>
> This is different from the previous one as it will FORCE lowercase in the rest of the word. <br>
> Supports PlaceholderAPI but requires `{ }` instead of `% %`.<br>
>
> **Example:** %capitalize_sentence-case-forced_{worldguard_region_name}%<br>
> **Output:** As the placeholder returns the name of the WorldGuard region "the_market_WORLD" it will return: `The market world`

> ### %capitalize_lowercase_{THE_PLACEHOLDER}%
> Allows you to use convert the output of the placeholder using lowercase. <br>
> This means that the whole parsed placeholder will be lowercase. <br>
> Supports PlaceholderAPI but requires `{ }` instead of `% %`.<br>
>
> **Example:** %capitalize_lowercase_{worldguard_region_name}%<br>
> **Output:** As the placeholder returns the name of the WorldGuard region "the_market_WORLD" it will return: `the market world`

> ### %capitalize_uppercase_{THE_PLACEHOLDER}%
> Allows you to use convert the output of the placeholder using uppercase. <br>
> This means that the whole parsed placeholder will be uppercase. <br>
> Supports PlaceholderAPI but requires `{ }` instead of `% %`.<br>
>
> **Example:** %capitalize_uppercase_{worldguard_region_name}%<br>
> **Output:** As the placeholder returns the name of the WorldGuard region "the_market_WORLD" it will return: `THE MARKET WORLD`
